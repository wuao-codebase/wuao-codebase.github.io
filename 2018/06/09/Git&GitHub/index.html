Git&GitHub 分布式版本管理工具

工具功能

  1. 分布式
    • 所有本地项目和服务器项目都保存着完整的版本信息。(集中式svn只在服务器保存版本信息)
  2. 版本管理
    • Git采用文件系统快照的方式，在保存每一个版本时做到不保存重复数据。
  3. 协同开发
    • Git对代码内容进行管理，允许多人同时对同一个文件进行修改。
  4. 权限管理
    • 团队内部人员权限控制。
    • 团队外部人员提交代码进行审核。
  5. 数据备份
    • 自动保存每次提交历史和文件\夹状态。
  6. 历史记录
    • 查看修改信息(人，时间，内容，日志)。
    • 恢复本地项目到某一历史时刻。
  7. 分支管理
    • 允许多分支同时开发。

Git安装

官网下载，安装基本上就是一直下一步，安装路径一般设为非中文路径。







验证成功：右击->Git Base Here->输入 git -version。以后这就是Git工作终端。

Git命令

Git集成了Linux命令，用着很方便。

初始化本地仓库

  • 本地仓库也就是我们项目，在项目根目录(如F:\workspace_idea\gittest)下打开Git终端

    注意：.git包含本地项目的相关信息，不要乱动

设置签名

  • 用于区别开发人员身份。

  • 项目级别签名(局部设置，优先级高于用户级签名，保存在.git/config中)

  • 用户级别签名(一般使用全局设置就可以了，保存在~/.gitconfig)。

    注意：这里的用户名和邮箱和Github账号不同，可以使用不一样的。

操作命令

  • git status 查看工作区，暂存区状态，如文件提交情况

  • git add [file name] 添加新建/修改的文件到暂存区

  • git rm –cached [file name] 删除暂存区文件

  • git commit [file name]” 提交缓存区文件到本地仓库

  • git commit [file name]” 提交缓存区文件到本地仓库

  • git log 查看历史信息

    注意：commit由哈希算法生成的版本唯一索引值，也是版本管理的关键。

  • git reset 版本控制
