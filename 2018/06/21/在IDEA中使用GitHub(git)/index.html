在IDEA中使用GitHub(git)

发表于 2018-06-21 | 分类于 工具 | 热度: ℃
字数统计: 384 | 阅读时长 ≈ 1

觉得使用Git命令行工具太麻烦了，那就来使用IEDA集成Git功能吧(>▽<)

准备工作

  • 安装Git，参考Git&GitHub。
  • 在IDEA中创建一个项目。

配置Git和GitHub

打开IDEA的设置，如图Default Settings为全局设置，Settings为项目设置。一般在项目设置中进行设置，我就单账号可以在全局设置中进行设置了。

配置Git

找到Git的安装目录，点击Test。成功！

配置GitHub

填写账号，密码，点击Test。成功！

首次上传项目

  • 首次上传说明GitHub上没有相应的仓库，因此这一步主要就是创建仓库。

  • 填写仓库名，地址别名，仓局描述

  • 选择要上传的文件，填写提交日志。等待上传成功就可以在网页看到了。

功能区

  • 基本功能，克隆，版本管理，冲突合并，分支管理等git功能。
  • 常用功能，依次是拉取远程仓库，提交项目，版本比较，历史版本，回退。

常规提交

  • 创建和修改一些文件。红色是未提交过的，蓝色是修改的。
  • 填写提交日志，选择commit and push，将提交到本地仓库，并准备推送到远程仓库。
  • 提交到远程仓库。

IDEA中的.gitignore功能。

  • IDEA中实现了.gitignore功能，可以自由选择不进行版本管理的文件。被忽略的文件显示为黄色。
    • 选择忽略的文件
    • 选择忽略的文件夹
    • 通配符选择文件。
